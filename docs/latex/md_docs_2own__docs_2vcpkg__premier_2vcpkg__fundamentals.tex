\begin{DoxyItemize}
\item {\ttfamily vcpkg} is a cross-\/platform package manager for C and C++ libraries. It simplifies the process of acquiring and building open-\/source libraries on Windows, mac\+OS, and Linux. The tool offers a collection of pre-\/built libraries and the capability to compile libraries from source. It\textquotesingle{}s particularly useful for C++ developers, as managing dependencies can often be a complex task in the C++ ecosystem.
\end{DoxyItemize}\hypertarget{md_docs_2own__docs_2vcpkg__premier_2vcpkg__fundamentals_autotoc_md254}{}\doxysubsection{\texorpdfstring{How to use vcpkg}{How to use vcpkg}}\label{md_docs_2own__docs_2vcpkg__premier_2vcpkg__fundamentals_autotoc_md254}

\item {\bfseries{Installation\+:}}

\end{DoxyCode}

\item Bootstrap the tool on Windows\+:

\end{DoxyCode}

\end{DoxyItemize}
\item {\bfseries{Installing Packages\+:}} To install a library, you can use\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{./vcpkg\ install\ <library-\/name>}

\end{DoxyCode}

\item {\bfseries{Integration\+:}}
\begin{DoxyItemize}
\item {\bfseries{Globally\+:}} If you want to use the libraries system-\/wide, you can integrate {\ttfamily vcpkg} with your build system. For instance, with Visual Studio on Windows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{.\(\backslash\)vcpkg\ integrate\ install}

\end{DoxyCode}


This command provides a system-\/wide integration. After running it, any MSBuild project will be able to find and link against libraries installed by {\ttfamily vcpkg}.
\item {\bfseries{Locally\+:}} For a local project, you typically pass the toolchain file to your build system. The toolchain file is located at {\ttfamily \texorpdfstring{$<$}{<}vcpkg\+\_\+root\texorpdfstring{$>$}{>}/scripts/buildsystems/vcpkg.cmake}. For instance, with CMake, you can specify the toolchain file as\+:

{\ttfamily bash cmake -\/B \mbox{[}build directory\mbox{]} -\/S . -\/DCMAKE\+\_\+\+TOOLCHAIN\+\_\+\+FILE=\mbox{[}vcpkg root\mbox{]}/scripts/buildsystems/vcpkg.cmake }
\end{DoxyItemize}
