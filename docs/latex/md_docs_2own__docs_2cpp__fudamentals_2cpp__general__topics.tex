The {\ttfamily static} keyword in C++ has several different uses depending on the context it\textquotesingle{}s used in. Here are the primary uses of the {\ttfamily static} keyword\+:


\begin{DoxyEnumerate}
\item {\bfseries{Static Variables in Functions}}\+:
\begin{DoxyItemize}
\item When used inside a function, the {\ttfamily static} keyword applied to a local variable makes that variable retain its value across multiple function calls. It\textquotesingle{}s initialized only once, when the program starts, and its value persists between function calls.
\end{DoxyItemize}
\item {\bfseries{Static Variables in Classes/\+Structs}}\+:
\begin{DoxyItemize}
\item Inside a class or struct, the {\ttfamily static} keyword applied to a member variable makes it shared among all instances of that class or struct. It\textquotesingle{}s also sometimes referred to as a "{}class variable"{} or "{}static member."{}
\end{DoxyItemize}
\item {\bfseries{Static Member Functions}}\+:
\begin{DoxyItemize}
\item A member function declared as {\ttfamily static} belongs to the class itself, not to any instance of the class. It can be called without creating an instance of the class and operates on class-\/level data.
\end{DoxyItemize}
\item {\bfseries{Static Local Functions}}\+:
\begin{DoxyItemize}
\item A local function declared as {\ttfamily static} is only accessible within the translation unit (source file) it\textquotesingle{}s defined in. It can\textquotesingle{}t be linked to from other translation units.
\end{DoxyItemize}
\item {\bfseries{Static Data Members in Classes (C++17 and later)}}\+:
\begin{DoxyItemize}
\item Starting from C++17, static data members of classes can be defined inside the class and must be initialized outside the class definition. They are shared among all instances of the class and can be useful for maintaining class-\/level information.
\end{DoxyItemize}
\item {\bfseries{Static Assertions (C++11 and later)}}\+:
\begin{DoxyItemize}
\item In C++11 and later, the {\ttfamily static\+\_\+assert} keyword is used to perform compile-\/time assertions. It helps ensure that certain conditions are met at compile time, or it triggers a compilation error.
\end{DoxyItemize}
\item {\bfseries{Static Initialization Order (C++ Static Initialization Fiasco)}}\+:
\begin{DoxyItemize}
\item The order in which global or static objects are initialized across different translation units isn\textquotesingle{}t guaranteed. This can lead to initialization order problems, often referred to as the "{}static      initialization order fiasco."{} Techniques like the Singleton pattern or using local static variables inside functions can help mitigate these issues.
\end{DoxyItemize}
\end{DoxyEnumerate}

These are some of the main uses of the {\ttfamily static} keyword in C++. Each use serves a specific purpose and contributes to different aspects of the language, such as variable lifetime, member functions, memory management, and compile-\/time checks.\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2cpp__general__topics_autotoc_md57}{}\doxysection{\texorpdfstring{Is it bad to intialize variables with Null inside the function signature?}{Is it bad to intialize variables with Null inside the function signature?}}\label{md_docs_2own__docs_2cpp__fudamentals_2cpp__general__topics_autotoc_md57}
In C++, it\textquotesingle{}s perfectly fine to provide default values for function parameters. This allows for a function to be called with fewer arguments than it has parameters, and in such cases, the missing arguments are initialized to their corresponding default values.

But, it\textquotesingle{}s important to note that the way you provide these default values might differ based on the type of parameter. For instance, you would typically initialize a pointer with {\ttfamily nullptr} (not {\ttfamily NULL}, which is more of a C-\/style), an integer with 0, and so on.

Here\textquotesingle{}s an example\+:

\DoxyCodeLine{\ \ \ \ \textcolor{comment}{//\ Your\ code\ here}}
\DoxyCodeLine{\}}

\end{DoxyCode}


In this case, if you call {\ttfamily foo()} without any arguments, {\ttfamily ptr} will be {\ttfamily nullptr} and {\ttfamily val} will be {\ttfamily 0}. Also, while this can make your code more flexible, it can also make it harder to understand if not used judiciously, as it\textquotesingle{}s not immediately clear when looking at a function call what values the missing arguments will have. The {\ttfamily static} keyword in C++ has different meanings depending on the context in which it\textquotesingle{}s used.

{\bfseries{1. Inside a function\+:}} If a variable is declared as {\ttfamily static} inside a function, it maintains its value between function calls. The {\ttfamily itoa} function is a standard library function in C (not included in C++) that converts an integer to a string. {\ttfamily itoa} stands for "{}integer to ASCII"{}. It\textquotesingle{}s often found in older C code and non-\/standard C++ code. Its prototype is\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{char}*\ \mbox{\hyperlink{class_vector__2d}{itoa}}(\textcolor{keywordtype}{int}\ \mbox{\hyperlink{_s_d_l__opengl__glext_8h_ac783c2a5a0d42f36a33959f6a34ae0ba}{value}},\ \textcolor{keywordtype}{char}*\ str,\ \textcolor{keywordtype}{int}\ base);}

\end{DoxyCode}


Here, {\ttfamily value} is the integer that you want to convert to a string. {\ttfamily str} is the string buffer where the converted string will be stored, and {\ttfamily base} is the number base for the conversion.

For equivalent functionality in C++, you can use the {\ttfamily std\+::to\+\_\+string} function, which converts various data types to a string, like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{std::string\ str\ =\ std::to\_string(12345);}

\end{DoxyCode}


Similar to {\ttfamily itoa}, there are a number of other functions that convert different types to strings in C\+:


\begin{DoxyEnumerate}
\item {\ttfamily ltoa}\+: Converts a long integer to a string.
\item {\ttfamily ultoa}\+: Converts an unsigned long integer to a string.
\item {\ttfamily lltoa}\+: Converts a long long integer to a string.
\item {\ttfamily ulltoa}\+: Converts an unsigned long long integer to a string.
\end{DoxyEnumerate}

In C++, you can use {\ttfamily std\+::to\+\_\+string} for these conversions as well. It is overloaded to support {\ttfamily int}, {\ttfamily long}, {\ttfamily long long}, {\ttfamily unsigned long}, {\ttfamily unsigned long long}, {\ttfamily float}, {\ttfamily double}, and {\ttfamily long double}.
