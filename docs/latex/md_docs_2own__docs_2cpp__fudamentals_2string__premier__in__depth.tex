Let\textquotesingle{}s delve into string literals and character literals in C++.\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md185}{}\doxysubsection{\texorpdfstring{Character Literals\+:}{Character Literals:}}\label{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md185}
A character literal represents a single character and is enclosed in single quotes. In C++, a character literal has type {\ttfamily char}.

Examples\+:


\begin{DoxyItemize}
\item `\textquotesingle{}a'{\ttfamily  -\/}\textquotesingle{}1\textquotesingle{}{\ttfamily  -\/}\textquotesingle{}\textquotesingle{}\`{}

There are also some special character literals called escape sequences\+:


\begin{DoxyItemize}
\item `'~\newline
\textquotesingle{}{\ttfamily \+: Newline -\/}\textquotesingle{}\textbackslash{}t\textquotesingle{}{\ttfamily \+: Tab -\/}\textquotesingle{}\textbackslash{}r\textquotesingle{}{\ttfamily \+: Carriage return -\/}\textquotesingle{}\textbackslash{}\textquotesingle{}\textquotesingle{}{\ttfamily \+: Single quote -\/}\textquotesingle{}"{}\textquotesingle{}{\ttfamily \+: Double quote -\/}\textquotesingle{}\textbackslash{}0\textquotesingle{}{\ttfamily \+: Null character -\/}\textquotesingle{}{\bfseries{}}\textquotesingle{}\`{}\+: Backspace

\begin{DoxyItemize}
\item {\ttfamily wchar\+\_\+t} is used for wide character literals and is prefixed by {\ttfamily L}, e.\+g., `L\textquotesingle{}Z'{\ttfamily .}
\item {\ttfamily C++11 introduced}char16\+\_\+t{\ttfamily for UTF-\/16 encoded characters with prefix}u{\ttfamily , e.\+g.,}u\textquotesingle{}Z\textquotesingle{}{\ttfamily and}char32\+\_\+t{\ttfamily for UTF-\/32 encoded characters with prefix}U{\ttfamily , e.\+g.,}U\textquotesingle{}Z\textquotesingle{}\`{}.
\end{DoxyItemize}\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md187}{}\doxysubsection{\texorpdfstring{String Literals\+:}{String Literals:}}\label{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md187}
A string literal is a sequence of characters surrounded by double quotes. The compiler appends a null character (`'\textbackslash{}0\textquotesingle{}\`{}) to every string literal, making it a null-\/terminated string.

Examples\+:


\begin{DoxyItemize}
\item {\ttfamily "{}\+Hello, World!"{}}
\item {\ttfamily "{}\+C++ Programming"{}}
\item {\ttfamily "{}12345"{}}
\end{DoxyItemize}

Similar to character literals, string literals also support escape sequences\+:


\begin{DoxyItemize}
\item {\ttfamily "{}\+This is a string with a newline character.\textbackslash{}n"{}}
\end{DoxyItemize}\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md188}{}\doxysubsubsection{\texorpdfstring{Raw String Literals\+:}{Raw String Literals:}}\label{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md188}
C++11 introduced raw string literals that allow you to write string literals without escaping characters. This is particularly useful for regex patterns, path specifications, or any multi-\/line string content. They are denoted by {\ttfamily R} and surrounded by {\ttfamily "{}( )"{}}.

Example\+:


\begin{DoxyItemize}
\item {\ttfamily R"{}(\+This is a "{}raw"{} string. No need to escape the "{}double quotes"{}.)"{}}
\item {\ttfamily R"{}(\+C\+:\textbackslash{}\+Path\textbackslash{}\+To\textbackslash{}\+Directory)"{}} (notice no need to double backslash)
\end{DoxyItemize}\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md189}{}\doxysubsubsection{\texorpdfstring{Wide and Unicode String Literals\+:}{Wide and Unicode String Literals:}}\label{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md189}

\begin{DoxyItemize}
\item Wide string literals use {\ttfamily wchar\+\_\+t} and are prefixed by {\ttfamily L}, e.\+g., {\ttfamily L"{}\+Hello"{}}.
\item UTF-\/8 string literals (C++11) use {\ttfamily char} and are prefixed by {\ttfamily u8}, e.\+g., {\ttfamily u8"{}\+Hello"{}}.
\item UTF-\/16 string literals use {\ttfamily char16\+\_\+t} and are prefixed by {\ttfamily u}, e.\+g., {\ttfamily u"{}\+Hello"{}}.
\item UTF-\/32 string literals use {\ttfamily char32\+\_\+t} and are prefixed by {\ttfamily U}, e.\+g., {\ttfamily U"{}\+Hello"{}}.
\end{DoxyItemize}\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md190}{}\doxysubsection{\texorpdfstring{Distinction Between String Literals and $<$tt$>$std\+::string$<$/tt$>$\+:}{Distinction Between String Literals and <tt>std::string</tt>:}}\label{md_docs_2own__docs_2cpp__fudamentals_2string__premier__in__depth_autotoc_md190}

\begin{DoxyItemize}
\item {\bfseries{C-\/style strings}}\+: Are arrays of characters terminated by a null character (`'\textbackslash{}0\textquotesingle{}\`{}). They require manual memory management and can lead to buffer overflows if not handled correctly.
\item {\bfseries{std\+::string}}\+: A safer and more convenient way to work with strings in C++. It manages memory automatically and provides a range of useful string manipulation functions.
\item {\bfseries{String Literal}}\+: Is a sequence of characters between double quotes. They are stored in a static memory region, and thus, are immutable. Attempts to modify them result in undefined behavior.
\item {\bfseries{Raw String Literal}}\+: Provides a way to represent strings without escaping. Useful for representing paths, regex, or multi-\/line strings.
\item {\bfseries{std\+::stringstream}}\+: A stream-\/based way to construct and manipulate strings. Especially useful when combining different data types into a single string.
\item Always remember that for heap-\/allocated variables, it\textquotesingle{}s necessary to release the memory using {\ttfamily delete} (or {\ttfamily delete\mbox{[}\mbox{]}} for arrays) to avoid memory leaks.

Here, {\ttfamily argv} is an array of pointers to {\ttfamily char}, where each pointer points to a null-\/terminated string (a C-\/style string). \begin{DoxyItemize}
\item {\ttfamily char\texorpdfstring{$\ast$}{*} argv\mbox{[}\mbox{]}} is an array of {\ttfamily char} pointers.
\item {\ttfamily char\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} argv} is a pointer to a {\ttfamily char} pointer, and it can also point to an array of {\ttfamily char} pointers.
\end{DoxyItemize}
