{\bfseries{Key Takeaways}}\+:


\begin{DoxyEnumerate}
\item {\bfseries{Read {\ttfamily const} with Pointers Right-\/to-\/\+Left}}\+: For example, {\ttfamily const int\texorpdfstring{$\ast$}{*} ptr} reads as "{}ptr is a pointer to an int that is constant"{}.
\item {\bfseries{Member Functions}}\+: Use {\ttfamily const} for member functions to indicate they won\textquotesingle{}t change the state of the object.
\item {\bfseries{Method Arguments}}\+: When passing by reference, use {\ttfamily const} if you don\textquotesingle{}t intend to modify the argument to prevent accidental modifications.
\item {\bfseries{Iterators}}\+: Using {\ttfamily const} iterators is a good way to ensure that you don\textquotesingle{}t inadvertently modify container elements.
\end{DoxyEnumerate}

Applying the {\ttfamily const} keyword properly in C++ can lead to safer and more readable code.\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2constants__in__depth_autotoc_md45}{}\doxysection{\texorpdfstring{Examples}{Examples}}\label{md_docs_2own__docs_2cpp__fudamentals_2constants__in__depth_autotoc_md45}
When you use {\ttfamily const} before a variable declaration in C++, it means that you can\textquotesingle{}t modify the variable\textquotesingle{}s value after it\textquotesingle{}s set. In the case of {\ttfamily const char\texorpdfstring{$\ast$}{*}}, you\textquotesingle{}re saying that the characters being pointed to cannot be modified, but the pointer itself can be.

In your example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{const}\ \textcolor{keywordtype}{char}\ *\mbox{\hyperlink{class_vector__2d}{my\_string}}\ =\ \textcolor{stringliteral}{"{}This\ is\ as\ close\ to\ a\ string\ primitive\ as\ you\ can\ get"{}};}

\end{DoxyCode}



\begin{DoxyItemize}
\item {\ttfamily my\+\_\+string} is a pointer to a constant character (or an array of characters in this case, which is a string literal). This means that you can\textquotesingle{}t use {\ttfamily my\+\_\+string} to modify the characters it points to\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{class_vector__2d}{my\_string}}[0]\ =\ \textcolor{charliteral}{'t'};\ \textcolor{comment}{//\ This\ is\ an\ error.}}

\end{DoxyCode}



\begin{DoxyItemize}
\item However, you can change what {\ttfamily my\+\_\+string} points to\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{class_vector__2d}{my\_string}}\ =\ \textcolor{stringliteral}{"{}This\ is\ another\ string"{}};}

\end{DoxyCode}


Now, {\ttfamily my\+\_\+string} points to a different string literal.


\begin{DoxyItemize}
\item If you want to make sure that the pointer itself can\textquotesingle{}t be changed either, you\textquotesingle{}d put the {\ttfamily const} after the {\ttfamily \texorpdfstring{$\ast$}{*}}\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{char}\ \textcolor{keyword}{const}\ *\ \mbox{\hyperlink{class_vector__2d}{my\_string}}\ =\ \textcolor{stringliteral}{"{}This\ is\ as\ close\ to\ a\ string\ primitive\ as\ you\ can\ get"{}};}

\end{DoxyCode}



\begin{DoxyItemize}
\item Now, you can\textquotesingle{}t change the characters that {\ttfamily my\+\_\+string} points to, and you can\textquotesingle{}t change what {\ttfamily my\+\_\+string} points to.
\item It\textquotesingle{}s worth noting that string literals are stored in read-\/only memory in C++, so attempting to modify them (without the {\ttfamily const} keyword) is undefined behavior, meaning it may work on some compilers and crash on others. It\textquotesingle{}s good practice to always use {\ttfamily const} with string literals to avoid this undefined behavior.

\begin{DoxyItemize}

\item You cannot modify the value of {\ttfamily YEAR} itself (i.\+e., you cannot make it point to a different memory location).

In summary\+:


\begin{DoxyItemize}
\item {\ttfamily NAME} and {\ttfamily BOOK} are both constant pointers to constant characters, so you cannot modify the memory they point to, and you cannot change their values.
\item {\ttfamily YEAR} is a constant pointer to a character, so you can modify the memory it points to, but you cannot change its value (i.\+e., you cannot make it point to a different memory location).
\end{DoxyItemize}
