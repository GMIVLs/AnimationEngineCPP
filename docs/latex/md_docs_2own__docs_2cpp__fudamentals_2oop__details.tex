{\bfseries{Notes}}\+:


\begin{DoxyItemize}
\item {\bfseries{Encapsulation}}\+: One of the key features of classes, allowing data (attributes) and functions (methods) to be bundled together. Using private and protected members helps in achieving this.
\item {\bfseries{Polymorphism}}\+: Through the use of virtual functions, C++ enables dynamic polymorphism where the correct function related to the derived class object is called even when accessed using a base class pointer/reference.
\item {\bfseries{Inheritance}}\+: Allows for creating a new class that is based on an existing class. The new class inherits attributes and behavior (methods) from the existing class.
\item {\bfseries{Friend Functions}}\+: Though it breaks the encapsulation principle, it\textquotesingle{}s sometimes useful for operator overloading or certain specific scenarios where an external function needs to access the private members.
\item {\bfseries{Explicit Constructors}}\+: Useful to avoid accidental type conversions that might lead to bugs.
\end{DoxyItemize}

When designing classes in C++, always aim for strong encapsulation, clear interfaces, and prefer composition over inheritance when extending functionalities.\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2oop__details_autotoc_md141}{}\doxysubsection{\texorpdfstring{Common Code Practice}{Common Code Practice}}\label{md_docs_2own__docs_2cpp__fudamentals_2oop__details_autotoc_md141}

\hypertarget{md_docs_2own__docs_2cpp__fudamentals_2oop__details_autotoc_md147}{}\doxysection{\texorpdfstring{The Rule of 5 in Class}{The Rule of 5 in Class}}\label{md_docs_2own__docs_2cpp__fudamentals_2oop__details_autotoc_md147}
The Rule of 5 is a guideline in C++ programming that suggests that if a class defines any of the following, it should define all five of them to ensure resource management is handled correctly and predictably\+:


\begin{DoxyEnumerate}
\item {\bfseries{Destructor}}\+: Cleans up resources when an object is destroyed.
\item {\bfseries{Copy Constructor}}\+: Constructs a new object as a copy of an existing object.
\item {\bfseries{Copy Assignment Operator}}\+: Assigns the contents of an existing object to another object.
\item {\bfseries{Move Constructor}}\+: Constructs a new object by acquiring the resources of an existing object, instead of copying them.
\item {\bfseries{Move Assignment Operator}}\+: Assigns the resources of an existing object to another object, instead of copying them.
\end{DoxyEnumerate}
