\item Any use of {\bfseries{reinterpret\+\_\+cast}}. Pretty much the only thing you\textquotesingle{}re allowed to do with an object that you got from reinterpret cast is to reinterpret cast it back to the original type. Almost everything else is undefined behavior, and yes the same goes for C-\/style casting. Famous Quake three algorithm for computing the inverse square root, reinterpreting the bytes of a float as a long is actually undefined behavior. You\textquotesingle{}re always allowed to reinterpret cast the address of an object as a {\bfseries{character type}}. This allows you to inspect or print out the bytes taht make up the object, so you cas see its memory laout. However, {\bfseries{as of C++ 20}}, this use of reinterpret cast is also not necessary, {\bfseries{bit\+\_\+cast}} interprets the bytes of one object as a different type. In this case, I can cast any object to an array of bytes of the same size.
\end{DoxyEnumerate}

\begin{quote}
Consider not use \sout{interpret\+\_\+cast} instead, use it to cast an object to char or use bit\+\_\+cast introduced in c++ 20. \end{quote}
