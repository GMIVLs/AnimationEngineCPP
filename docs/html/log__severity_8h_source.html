33// The recommended semantics of the log levels are as follows:
34//
35// INFO:
36// Use for state changes or other major events, or to aid debugging.
37// WARNING:
38// Use for undesired but relatively expected events, which may indicate a
39// problem
40// ERROR:
41// Use for undesired and unexpected events that the program can recover from.
42// All ERRORs should be actionable - it should be appropriate to file a bug
43// whenever an ERROR occurs in production.
44// FATAL:
45// Use for undesired and unexpected events that the program cannot recover
46// from.
47
48// Variables of type LogSeverity are widely taken to lie in the range
49// [0, NUM_SEVERITIES-1]. Be careful to preserve this assumption if
50// you ever need to change their values or add a new severity.
51typedef int LogSeverity;
52

72// NDEBUG usage helpers related to (RAW_)DCHECK:
73//
74// DEBUG_MODE is for small !NDEBUG uses like

77// #ifndef NDEBUG
